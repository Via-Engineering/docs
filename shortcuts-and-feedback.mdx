Via BOM includes keyboard shortcuts designed for heavy search and review workflows, and a built-in feedback flow for reporting wrong results or inaccurate part data. Using both makes your workflow faster and helps the product improve.

## Keyboard Shortcuts

Via exposes a set of keyboard shortcuts aimed at keeping your hands off the mouse during intensive search and review sessions. The full set of currently available shortcuts is listed below.

| Shortcut | Action |
|---|---|
| `Mod+K` | Go home |
| `S` | Toggle the main sidebar |
| `I` | Toggle the info sidebar |
| `Mod+P` | Switch to projects |
| `Mod+L` | Switch to libraries |
| `V` | Open the revision picker in a project |
| `B` | Open the BOM picker in a project |
| `` ` `` | Create a new revision or BOM in project context |
| `Space` | Open project search or quick look, depending on context |
| `D` | Open the datasheet from quick look |
| `M` | Search the part on Mouser from quick look |

<Note>
  Shortcuts can change as the product evolves. Use the in-app settings view as
  the source of truth if a shortcut feels out of date.
</Note>

## Quick Look Workflow

Quick look is one of the best surfaces to lean on the keyboard. The most efficient loop looks like this:

1. Hover or select a result
2. Open quick look
3. Inspect the top specs
4. Open the datasheet if needed
5. Move to the next candidate without losing your place

This loop is often faster than drilling into each part page individually.

## Reporting Problems

The bug report flow is built into the app because product feedback is part of the working loop. Use it when:

- Search returned obviously wrong results
- Part data is inaccurate
- A datasheet link is broken or unrelated
- Import parsing is incorrect
- A workspace, project, library, or ruleset flow feels unreliable

## What Makes a Good Report

<Tip>
  The most useful reports include:

  - The exact query or part number
  - What Via showed
  - What you expected instead
  - Whether the issue blocks a real decision

  Via can attach current part data or search context to your report automatically — that's why reporting from the exact screen where the problem occurred is more useful than sending a vague note later.
</Tip>
