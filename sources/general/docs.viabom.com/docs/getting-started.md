# Getting Started

Copy page

Get into Via quickly: sign in, choose a workspace, run a search, and create your first project.

Via is built for hardware engineers who need to move from "find the right part" to "use it in a real BOM" without jumping between too many tools.

## [Before You Start](https://docs.viabom.com/#before-you-start)

- You need an active Via account and access to the right workspace.
- Search coverage, extracted data, and ranking continue to improve, so confirm final selections on the part page and datasheet when the decision matters.
- The changelog and status page are worth keeping close at hand as the product evolves.

## [1\. Sign In](https://docs.viabom.com/#1-sign-in)

Use the normal login flow from the app home page. After sign-in, Via creates or loads your account data and then loads your available workspaces.

If you do not see your expected workspace yet, check that you are using the right email and then ask your team admin for an invitation.

## [2\. Pick Your Workspace](https://docs.viabom.com/#2-pick-your-workspace)

The left sidebar starts with the workspace switcher. Your active workspace controls what shared projects, libraries, and rulesets you see.

Use it to:

- switch between organizations
- create a new workspace
- join an existing workspace
- open workspace settings when you need to manage branding, domains, or members

Workspace scope matters. If something seems missing, first confirm you are in the right workspace.

## [3\. Run Your First Search](https://docs.viabom.com/#3-run-your-first-search)

The fastest way to get oriented is to search for a real part or requirement:

- `100nF 0402 capacitor`
- `INA228 current sense monitor`
- `low noise 3.3V LDO`

Via search supports part numbers, package sizes, values, and plain-language requirements in the same box.

If you already know what project you are working on, open that project first and search from there so Via can use your current BOM and active ruleset as context.

## [4\. Inspect a Part Before Committing](https://docs.viabom.com/#4-inspect-a-part-before-committing)

Open a result's part details to answer questions like:

- What did Via extract from the datasheet?
- Is the datasheet linked and readable?
- Are there nearby variants in the same series?
- Do distributor details look current enough for this decision?
- Did Via attach design calculators for this part?

If the data looks off, report it directly from the product so the team can review the exact context.

## [5\. Create Your First Project](https://docs.viabom.com/#5-create-your-first-project)

From the signed-in home screen, use **Create Project** to start a new project workspace. A project is where Via keeps your selected parts, revisions, and BOMs together.

Inside a project you can:

- add parts from search
- import BOM files
- create new revisions
- create multiple BOMs within a revision
- export the current BOM when you need to hand it off

If you already have a starting spreadsheet, go straight to the BOM import flow.

## [6\. Save Reusable Preferences](https://docs.viabom.com/#6-save-reusable-preferences)

As you settle on preferred parts and constraints:

- use **Libraries** to keep preferred components close at hand
- use **Rulesets** to encode supply-chain, lifecycle, cost, or qualification preferences that should influence ranking automatically

Both are versioned so you can evolve them without losing prior working states.
