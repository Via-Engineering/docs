# Libraries

Copy page

Save preferred parts into reusable libraries so future searches can lean toward parts your team already trusts.

Libraries let you keep a reusable set of preferred components outside any single project.

## [What A Library Is For](https://docs.viabom.com/#what-a-library-is-for)

Use a library when you want to preserve a set of parts that are:

- already approved by your team
- preferred for sourcing or qualification reasons
- good defaults for recurring designs
- worth biasing search toward in future work

Libraries are especially helpful when you want continuity across multiple projects without cloning old BOMs by hand.

## [Create And Populate A Library](https://docs.viabom.com/#create-and-populate-a-library)

Libraries live in the left sidebar alongside projects and rulesets.

Typical flow:

1. create a new library
2. open it
3. search for parts you trust
4. add those parts into the active library version

You can build a library gradually. It does not need to be complete before it starts being useful.

## [Versioning](https://docs.viabom.com/#versioning)

Libraries are versioned so you can revise your preferred set without overwriting the previous working state.

Use a new library version when:

- your preferred manufacturer changes
- you want a cleaner shortlist for a new product line
- you need to retire or replace parts without losing prior choices

## [How Libraries Affect Search](https://docs.viabom.com/#how-libraries-affect-search)

An active library can bias search toward:

- parts already present in the library
- close family matches
- manufacturers you already prefer

This is a bias, not an absolute filter. If search keeps steering too hard toward older choices, try searching outside that library context and compare the results.

## [Good Use Cases](https://docs.viabom.com/#good-use-cases)

- keep a house list of passives and common support ICs
- save vetted alternatives for supply-chain resilience
- preserve category-specific shortlists for repeated board families

## [Current Notes](https://docs.viabom.com/#current-notes)

- treat library bias as helpful guidance, not guaranteed policy
- keep versions clean when preferences change materially
- report bad library-biased matches if the app overfits to the saved set
