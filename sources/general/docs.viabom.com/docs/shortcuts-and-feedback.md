## [Useful Shortcuts](https://docs.viabom.com/#useful-shortcuts)

Via already exposes a set of keyboard shortcuts aimed at heavy search and review workflows.

Common ones include:

- `Mod+K`: go home
- `S`: toggle the main sidebar
- `I`: toggle the info sidebar
- `Mod+P`: switch to projects
- `Mod+L`: switch to libraries
- `V`: open the revision picker in a project
- `B`: open the BOM picker in a project
- `` ` ``: create a new revision or BOM in project context
-   `Space`: open project search or quick look, depending on context
-   `D`: open the datasheet from quick look
-   `M`: search the part on Mouser from quick look

Shortcuts can evolve, so use the in-app settings view as the source of truth if something feels out of date.

## [Quick Look Workflow](https://docs.viabom.com/#quick-look-workflow)

Quick look is one of the best places to lean on the keyboard:

- hover or select a result
- open quick look
- inspect the top specs
- open the datasheet if needed
- move to the next candidate without losing your place

That loop is often faster than fully drilling into each part page.

## [Reporting Problems](https://docs.viabom.com/#reporting-problems)

The bug report flow is built into the app because product feedback is part of the working loop.

Use it when:

- search returned obviously wrong results
- part data is inaccurate
- a datasheet link is broken or unrelated
- import parsing is incorrect
- a workspace, project, library, or ruleset flow feels unreliable

The most useful reports include:

- the exact query or part number
- what Via showed
- what you expected instead
- whether the issue blocks a real decision

## [What Gets Attached](https://docs.viabom.com/#what-gets-attached)

Depending on the surface, Via can attach current part data or search context to help the team reproduce the issue faster. That is why reporting from the exact screen where the problem happened is usually better than sending a vague note later.

## [Current Product Notes](https://docs.viabom.com/#current-product-notes)

Keep these in mind while using the app:

- component coverage is still incomplete
- search ranking still needs tuning in some categories
- extracted values can be missing or occasionally wrong
- import and comparison flows may need manual cleanup

Reporting those misses is part of how the product improves.
